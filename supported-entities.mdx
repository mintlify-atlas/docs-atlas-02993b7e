## Overview

Presidio includes predefined recognizers for a wide range of PII entity types. Each recognizer uses one or more detection methods:

| Method | Description |
| --- | --- |
| **Pattern match** | Regular expression patterns matched against input text |
| **NER (Named Entity Recognition)** | Machine learning models that identify entities by context and learned representations |
| **Checksum** | Algorithmic validation of digit sequences (e.g., Luhn for credit cards, IBAN checksums) |
| **Context** | Surrounding words and phrases that increase or decrease confidence in a detected span |
| **Custom logic** | Deterministic rules beyond simple regex, such as format validation or cross-field checks |

<Note>
  Presidio uses automated detection and cannot guarantee that all PII will be found. Additional systems and protections should be employed alongside Presidio.
</Note>

## Global entities

These recognizers are language-agnostic and apply across all supported locales.

| Entity type | Description | Detection method |
| --- | --- | --- |
| `CREDIT_CARD` | A credit card number between 12 and 19 digits. | Pattern match and checksum |
| `CRYPTO` | A crypto wallet number. Currently only Bitcoin addresses are supported. | Pattern match, context and checksum |
| `DATE_TIME` | Absolute or relative dates, periods, or times smaller than a day. | Pattern match and context |
| `EMAIL_ADDRESS` | An email address identifying an email box to which messages are delivered. | Pattern match, context and RFC-822 validation |
| `IBAN_CODE` | An International Bank Account Number (IBAN) used to identify bank accounts across national borders. | Pattern match, context and checksum |
| `IP_ADDRESS` | An Internet Protocol (IP) address — either IPv4 or IPv6. | Pattern match, context and checksum |
| `MAC_ADDRESS` | A Media Access Control (MAC) address uniquely assigned to a network interface. | Pattern match and context |
| `NRP` | A person's nationality, religious or political group. | Custom logic and context |
| `LOCATION` | Name of a politically or geographically defined location (cities, provinces, countries, bodies of water, mountains). | Custom logic and context |
| `PERSON` | A full person name, which can include first names, middle names or initials, and last names. | Custom logic and context |
| `PHONE_NUMBER` | A telephone number. | Custom logic, pattern match and context |
| `MEDICAL_LICENSE` | Common medical license numbers. | Pattern match, context and checksum |
| `URL` | A URL (Uniform Resource Locator) that uniquely identifies a resource on the internet. | Pattern match, context and top-level URL validation |

## Medical entities

Medical entities are detected using the `MedicalNERRecognizer`, which requires the `transformers` extra (`pip install presidio-analyzer[transformers]`). The recognizer uses the [blaze999/Medical-NER](https://huggingface.co/blaze999/Medical-NER) model by default.

<Note>
  The `MedicalNERRecognizer` is not loaded by default. You must explicitly add it to the `AnalyzerEngine` registry.
</Note>

| Entity type | Description | Detection method |
| --- | --- | --- |
| `MEDICAL_DISEASE_DISORDER` | A disease or disorder (e.g., diabetes, hypertension). | NER model (HuggingFace transformers) |
| `MEDICAL_MEDICATION` | A medication or drug name (e.g., metformin, aspirin). | NER model (HuggingFace transformers) |
| `MEDICAL_THERAPEUTIC_PROCEDURE` | A therapeutic or diagnostic procedure (e.g., surgery, MRI). | NER model (HuggingFace transformers) |
| `MEDICAL_CLINICAL_EVENT` | A clinical event (e.g., admission, discharge). | NER model (HuggingFace transformers) |
| `MEDICAL_BIOLOGICAL_ATTRIBUTE` | A biological attribute or measurement (e.g., blood pressure, BMI). | NER model (HuggingFace transformers) |
| `MEDICAL_BIOLOGICAL_STRUCTURE` | A biological or anatomical structure (e.g., liver, left ventricle). | NER model (HuggingFace transformers) |
| `MEDICAL_FAMILY_HISTORY` | A family medical history reference. | NER model (HuggingFace transformers) |
| `MEDICAL_HISTORY` | A patient medical history reference. | NER model (HuggingFace transformers) |

## Adding custom entities

Presidio is fully extensible. You can create new recognizers for entity types not listed here — including pattern-based recognizers, ML-based recognizers, and recognizers that call external services.

<Tip>
  See the [custom recognizers documentation](/customization/custom-recognizers) for step-by-step instructions on implementing and registering a new recognizer.
</Tip>
