## Overview

The samples directory contains Jupyter notebooks, Python scripts, and deployment templates covering the full range of Presidio capabilities. Use these as starting points for your own integration.

<Note>
  All samples are maintained in the [microsoft/presidio GitHub repository](https://github.com/microsoft/presidio/tree/main/docs/samples). Most Python notebooks can be run locally or in any hosted Jupyter environment.
</Note>

## Presidio CLI

The `presidio-cli` package provides a command-line interface for scanning files and directories for PII using Presidio Analyzer.

```bash
pip install presidio-cli
python -m spacy download en_core_web_lg

# Scan the current directory with default config
presidio .

# Scan with a specific config file
presidio -c my_config.yaml src/

# Scan a single file and output parsable JSON
presidio -d "entities:
  - PERSON
  - EMAIL_ADDRESS" -f parsable myfile.py
```

The CLI uses a `.presidiocli` YAML config file (or `-c` / `-d` flags) with these options:

| Option | Description |
|---|---|
| `language` | Language for detection (default: `en`) |
| `entities` | List of entity types to detect (default: all) |
| `ignore` | File/path patterns to skip (glob syntax) |
| `allow` | Tokens that should not be flagged as PII |

**Output formats**: `standard`, `github`, `colored`, `parsable`, `auto` (default)
